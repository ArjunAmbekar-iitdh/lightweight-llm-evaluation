llm-edge-bench 🚀Comparative performance analysis of ultra-lightweight LLMs: Cloud (Colab) vs. Edge (Jetson Xavier).This repository contains the benchmarking suite and implementation guides for evaluating <4B parameter Large Language Models. The project focuses on the trade-offs between cloud-based inference and local edge deployment on NVIDIA Jetson hardware.📊 Research ObjectivesThe goal of this project is to provide empirical data for a review paper comparing:Performance: Tokens Per Second (TPS) and Time to First Token (TTFT).Efficiency: VRAM consumption and Power Draw (Watts).Quantization: Impact of 4-bit (Q4_K_M) quantization on inference speed across platforms.💻 Target HardwareFeatureCloud Baseline (Colab)Edge Target (Jetson Xavier)GPUNVIDIA T4 / L4512-core Volta with Tensor CoresRAM12GB - 16GB GDDR68GB - 16GB (Shared)OSLinux (Ubuntu 22.04)Ubuntu (L4T / JetPack)Power~70W - 150W10W - 30W🧠 Supported ModelsGemma-3-1B / 4B (Google)Qwen-2.5-0.5B / 1.5B (Alibaba)Phi-4-mini (Microsoft)🛠️ Project Structure/colab: Notebooks for baseline testing and GGUF model dumping./jetson: Scripts for llama.cpp and jtop power monitoring./data: CSV exports of benchmark results./scripts: Python utilities for calculating $TPS$ and $Latency$.🚀 Quick Start (Phase 1: Colab)Open the notebook in the /colab folder.Run the benchmarking cells to establish baseline TPS.Use the export script to generate the .gguf files for the Xavier.📈 Key Metrics MeasuredThroughput: $Tokens / Second$Latency: $TTFT (ms)$Energy: $Joules / Token$ (Calculated via tegrastats on Xavier)Memory: Peak VRAM utilization
